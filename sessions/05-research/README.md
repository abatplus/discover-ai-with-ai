# Session 5 – Research with AI (Optional)

> **Practical tip:** Before committing to a library or architecture, let Claude research and compare options for you. You'll get a decision matrix in minutes instead of hours.

> **Important:** Start a fresh Claude Code session for this one. A new context keeps Claude focused.

**Goal:** Use Claude as a research assistant – web search, documentation lookup, decision preparation.

**Demo:**

> **Prompt** – copy the following into Claude Code:
```
Compare three popular logging libraries for this project's language.
Search the web for current recommendations.
Create a decision matrix with these criteria:
simplicity, performance, community support, configurability.
Give a recommendation with reasoning.
```

**Your turn:** Research a technical question from your real project. For example:
- "Which state management pattern fits our architecture?"
- "Compare migration paths from library X to Y"
- "What are current best practices for [topic]?"

**What you learned:**
- Claude can search the web and summarize information
- Decision matrices help make informed tech decisions
- You save hours of manual research
