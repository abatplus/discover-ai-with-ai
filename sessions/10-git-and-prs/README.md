# Session 10 – Git & PR Workflows (Optional)

> **Practical tip:** Let Claude write your commit messages and PR descriptions. They'll be better than what most devs write at 5 PM on a Friday.

> **Important:** Start a fresh Claude Code session for this one. A new context keeps Claude focused.

**Goal:** Claude takes over the tedious parts of Git – commit messages, PR descriptions, changelogs.

**Demo:**

> **Prompt** – copy the following into Claude Code:
```
Look at the current changes in the project (git diff, git log).
Write a meaningful commit message for all uncommitted changes.
Then create a PR description with summary, changes, and test plan.
```

**Your turn:** Try one of these:
- "Create a changelog for the last 5 commits"
- "I have a merge conflict – help me resolve it" (if you have one)
- "Write a commit message for these changes" (in your real project)

**What you learned:**
- Claude writes better commit messages than most devs at 5 PM
- PR descriptions cost you 10 seconds instead of 10 minutes
- Git workflows get faster, not just code
