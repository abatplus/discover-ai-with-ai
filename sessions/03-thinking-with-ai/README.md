# Session 3 – Thinking with AI

> **Practical tip:** The best code starts with good questions, not with typing. Let Claude interview you about your idea before writing a single line.

> **Important:** Start a fresh Claude Code session for this one. A new context keeps Claude focused.

**Goal:** Experience Claude as a thinking partner who collaboratively develops ideas with you – instead of jumping straight to code.

**Prerequisite:** Superpowers must be installed (Session 0).

**Demo:**

> **Prompt** – copy the following into Claude Code:
```
/superpowers:brainstorm
I want to brainstorm a filtering and sorting feature for the Task API.
Users should be able to filter tasks by status and priority,
and sort by different fields.
```

**What happens now:**
- Claude asks you questions instead of jumping to code
- It works out the requirements with you step by step
- It proposes different approaches and explains the trade-offs
- At the end, you get a well-thought-out design

**Your turn:** Brainstorm your own feature with `/superpowers:brainstorm`. Ideas:
- Tags/Labels for tasks
- Due dates with reminders
- Task comments
- Or something entirely your own

**Tips:**
- Feel free to say "No, I don't like that" or "Let's try a different approach"
- The brainstorming is a dialogue – not a monologue
- You don't have to implement the feature – it's about the thinking process

**What you learned:**
- Claude can think with you in a structured way, not just produce code
- Superpowers brainstorming prevents premature implementation
- Good software development starts with good questions, not code
